# Build 61 — Customer Collection Pass

Changed files:

- `src/OrderDetail.jsx`
- `src/CollectionPassPage.jsx`
- `src/ConnectedApp.jsx`
- `BUILD_61.md`

## Summary

Customer order detail now loads collection pass data from:

- `/api/internal/storefront/customer/orders/[id]/collection-pass`

If the order uses collection fulfilment, the customer sees:

- 6-digit collection PIN
- QR scan link
- location label
- location address
- pickup instructions
- collection truth note
- readiness state

A standalone scan page was added:

- `/collection-pass?token=...`

This page verifies the token through:

- `/api/internal/collection/verify?token=...`

The hosted route has been wired in `ConnectedApp.jsx` before normal storefront routing.

Build fingerprint:

`HOSTED-THEME-BUILD-61-COLLECTION-PASS-v2026-06-05`

## Notes

- This does not change checkout pricing or VAT logic.
- This does not create a duplicate order system.
- This uses the collection pass generated by the admin/backend Build 61 APIs.
- The QR visual is generated from the pass URL for launch testing; a later build can replace this with a self-hosted QR renderer.
